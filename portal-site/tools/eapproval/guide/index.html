기안부터 승인까지 실습하기

실습생과 지도자가 역할을 나눠 실제 결재 흐름을 순서대로 연습합니다.

권장 실습 시나리오

실습생: 문서 기안프로그램 계획서 또는 일반기안을 작성하고 결재를 요청합니다.
지도자: 검토와 반려문서를 확인한 뒤 구체적인 보완 의견과 함께 반려합니다.
실습생: 수정과 재기안반려 의견을 확인하고 문서를 수정한 뒤 다시 기안합니다.
지도자: 최종 승인보완된 내용과 기안차수를 확인하고 승인합니다.
실습생: 결과 확인완료문서에서 결재 결과와 의견을 확인하고 PDF로 출력합니다.

역할별 주요 기능

실습생실습지도자
기안·임시저장·결재요청결재대기 문서 검토
회수·수정·재기안승인·반려·의견 등록
내 기안함·완료문서 확인사용자와 기본 결재선 관리
게시글·댓글 작성설치 점검·백업·초기화

기안할 때 확인할 것

  • 문서종류와 보존연한이 실습 과제에 맞는지 확인합니다.
  • 결재자를 선택하고 순서를 확인합니다.
  • 제목과 본문에는 교육용 가상 내용을 사용합니다.
  • 실제 개인정보나 기관 내부자료를 첨부하지 않습니다.

교육 종료 후 백업·초기화

환경설정 → 교육환경 초기화에서 현재 자료를 별도 스프레드시트로 백업한 뒤 사용자·문서·결재이력·게시판·댓글·설정 자료를 초기화할 수 있습니다. 첨부파일 폴더는 Google Drive 휴지통으로 이동하며 실습지도자 계정은 유지됩니다.

초기화 전 백업 파일이 정상적으로 만들어졌는지 확인하고, 필요한 첨부파일은 별도로 보관하세요.

문제가 생겼을 때

  • 화면이 이전 상태라면 새 배포가 웹앱에 반영되었는지 확인합니다.
  • 필수 시트나 첨부폴더 오류는 설치 상태 점검의 복구 기능을 실행합니다.
  • 로그인 목록에 사용자가 없으면 기본 설정에서 계정이 활성 상태인지 확인합니다.
  • 기관 계정에서 배포 옵션이 제한되면 Google Workspace 관리자에게 문의합니다.
